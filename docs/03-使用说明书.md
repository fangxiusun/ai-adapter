# resp2chat 使用说明书

## 1. 安装

### 1.1 从源码构建

```bash
# 前置条件：Go ≥ 1.21
go version

# 克隆项目
cd ai-adapter

# 构建
go build -o resp2chat ./cmd/server/

# 验证
./resp2chat --help
```

### 1.2 下载预编译二进制

从 GitHub Releases 下载对应平台的二进制文件。

## 2. 快速开始

### 步骤 1：创建配置文件

```bash
cp config.example.yaml config.yaml
```

### 步骤 2：编辑配置

```yaml
# config.yaml
server:
  host: "0.0.0.0"
  port: 8080

channels:
  - id: "mimo"
    name: "MiMo"
    base_url: "https://api.xiaomimimo.com/v1"
    wire_api: "chat"
    enabled: true
    models:
      - id: "mimo-v2.5-pro"
        display_name: "MiMo V2.5 Pro"
        context_window: 1000000
    default_model: "mimo-v2.5-pro"
    keys:
      - value: "sk-your-key-here"
        name: "主 key"
    key_strategy: "round-robin"
```

### 步骤 3：启动代理

```bash
./resp2chat -config config.yaml
```

看到启动信息后，代理已运行在 `http://0.0.0.0:8080`。

### 步骤 4：测试


## 3. 配置详解

### 3.1 服务器配置

```yaml
server:
  host: "0.0.0.0"      # 监听地址（0.0.0.0 允许外部访问）
  port: 8080            # 监听端口
  api_token: ""         # API 认证 token（空=不认证）
```

### 3.2 日志配置

```yaml
logging:
  level: "info"         # debug | info | warn | error
  file: ""              # 日志文件路径（空=stdout）
  max_size_mb: 100      # 单文件最大大小
  max_backups: 3        # 保留备份数
  log_request_body: false  # 是否记录请求体（调试用）
  log_io: false         # 是否记录客户端输入输出（含SSE流式输出）
```

**日志级别说明**：
- `debug`：最详细，包含请求/响应体、工具映射、key 选择
- `info`：请求摘要、配置变更（推荐）
- `warn`：重试、key 暂停
- `error`：上游错误、翻译失败

**log_io 说明**：
- 开启后记录客户端发送的完整请求体（`client_input`）
- 开启后记录返回给客户端的完整响应体（`client_output`）
- 流式请求记录每个 SSE 事件（`sse_event`）
- 仅在 `level: debug` 时生效（因为使用 `Debug` 级别输出）

### 3.3 渠道配置

```yaml
channels:
  - id: "mimo"                    # 唯一标识
    name: "MiMo"                  # 显示名称
    base_url: "https://api.xiaomimimo.com/v1"  # 上游地址
    wire_api: "chat"              # 上游接口类型："chat" | "responses"
    enabled: true                 # 是否启用
    default_model: "mimo-v2.5-pro"  # 默认模型
```

#### 渠道模型配置

```yaml
    models:
      - id: "mimo-v2.5-pro"       # 模型 ID
        display_name: "MiMo V2.5 Pro"  # 显示名称
        context_window: 1000000    # 上下文窗口
        max_output_tokens: 131072  # 最大输出 token
        supports_images: false     # 是否支持图片
        supports_reasoning: true   # 是否支持推理
        aliases:                   # 模型别名
          - "mimo-pro"
```

#### Key 配置

```yaml
    keys:
      - value: "sk-xxx"           # API Key
        name: "主 key"            # Key 名称（用于显示）
      - value: "sk-yyy"
        name: "备用 key"
```

#### Key 策略

```yaml
    key_strategy: "round-robin"   # 轮转策略
```

| 策略 | 说明 |
|------|------|
| `round-robin` | 轮询（默认） |
| `random` | 随机选择 |
| `least-errors` | 选择错误最少的 key |
| `least-latency` | 选择平均延迟最低的 key |
| `least-rate-limited` | 选择近期被 429 限制最少的 key（推荐多 key 场景） |

#### 重试配置

```yaml
    max_retries: 3                # 最大重试次数
    retry_delay_ms: 500           # 重试间隔 (ms)
    request_timeout_ms: 60000     # 请求超时 (ms)
```

**自动重试行为**：
- **401 错误**：永久跳过该 key，切换到下一个 key（不重试同一 key）
- **429 错误**：跳过当前 key，等待递增延迟（500ms × attempt）后切换到下一个 key
- **5xx 错误**：跳过当前 key，切换到下一个 key
- **所有 key 都失败**：返回 503 错误给客户端

#### Fan-out 配置

```yaml
    fanout:
      enabled: false              # 是否启用
      count: 2                    # 同时发送的 key 数
      wait_all: false             # false=返回第一个成功; true=等所有完成
```

#### 思考模式配置

```yaml
    thinking:
      default_enabled: true       # 默认开启思考模式
      force_high_effort: false    # 强制高强度思考
```

#### 联网搜索配置

```yaml
    web_search:
      enabled: false              # 是否转发 web_search
```

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/responses` | Responses API 端点 |
| POST | `/v1/chat/completions` | Chat Completions API 端点 |

### 5.2 Admin API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | Web UI |
| GET | `/admin/api/health` | 健康检查 |
| GET | `/admin/api/stats` | 统计信息 |
| GET | `/admin/api/channels` | 渠道列表 |
| GET | `/admin/api/channels/:id` | 渠道详情 |

| GET | `/admin/api/config` | 当前配置 |

### 5.3 日志查询参数

```
GET /admin/api/logs?channel=mimo&statusMin=400&limit=50
```

| 参数 | 说明 |
|------|------|
| channel | 渠道 ID 过滤 |
| statusMin | 最小状态码 |
| statusMax | 最大状态码 |
| from | 起始时间戳 (ms) |
| to | 结束时间戳 (ms) |
| limit | 返回数量（默认 100） |
| offset | 偏移量 |

## 6. 错误处理

### 6.1 错误响应格式

```json
{
  "error": {
    "type": "error",
    "code": "no_channel",
    "message": "no channel found for model: xxx",
    "status": 404
  }
}
```

### 6.2 常见错误

| 错误代码 | 说明 | 解决方案 |
|----------|------|----------|
| `no_channel` | 未找到匹配的渠道 | 检查模型 ID 和渠道配置 |
| `missing_model` | 请求缺少 model 字段 | 在请求体中添加 model |
| `invalid_json` | 请求体 JSON 解析失败 | 检查请求体格式 |
| `upstream_error` | 上游返回错误（重试后仍失败） | 检查 API Key 和上游状态 |
| `all_retries_failed` | 所有重试都失败 | 检查所有 key 是否有效，查看日志 |
| `no_available_keys` | 所有 key 都不可用 | 可能所有 key 都被 401 永久跳过，需手动 Resume |
| `translate_failed` | 协议翻译失败 | 检查日志中的详细错误 |

### 6.3 自动重试行为

代理会自动处理以下上游错误，尽可能给用户返回正确结果：

| 错误类型 | 处理方式 | 日志行为 |
|----------|----------|----------|
| 401 Unauthorized | 永久跳过该 key，切换到下一个 key | 明文打印 key 名称和值 |
| 429 Rate Limited | 跳过当前 key，等待递增延迟后切换 | 记录 429 频率 |
| 5xx Server Error | 跳过当前 key，切换到下一个 key | 记录错误 |
| 网络错误 | 跳过当前 key，切换到下一个 key | 记录错误 |

重试延迟：`500ms × (attempt + 1)`（递增）

## 7. 故障排查

### 7.1 启动失败

**问题**：`load config: open config.yaml: no such file`

```bash
# 确认配置文件存在
ls -la config.yaml

# 使用绝对路径
./resp2chat -config /path/to/config.yaml
```

**问题**：`at least one channel is required`

```yaml
# 确保至少配置了一个渠道
channels:
  - id: "mimo"
    # ...
```

### 7.2 请求失败

**问题**：`all_retries_failed` 或日志显示 401

```bash
# 401 会被永久跳过，检查 API Key 是否有效
curl https://api.xiaomimimo.com/v1/models \
  -H "Authorization: Bearer sk-your-key"

# 在 Web UI 中 Resume 被跳过的 key
# 或重启代理重置所有 key 状态
```

**问题**：日志显示频繁 429

```yaml
# 增加 key 数量分散请求
keys:
  - value: "sk-key-1"
    name: "key-1"
  - value: "sk-key-2"
    name: "key-2"
  - value: "sk-key-3"
    name: "key-3"

# 使用 least-rate-limited 策略
key_strategy: "least-rate-limited"

# 增加重试次数
max_retries: 5
```

**问题**：`upstream error 400: model not found`

```yaml
# 确认模型 ID 正确
models:
  - id: "mimo-v2.5-pro"  # 必须与上游支持的模型 ID 一致
```

### 7.3 查看日志

```bash
# 开启详细日志
# 修改 config.yaml
logging:
  level: "debug"
  log_request_body: true

# 重启代理
./resp2chat -config config.yaml
```
