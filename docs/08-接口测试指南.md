# 接口测试指南

> **Windows 用户注意**：在 Windows 环境下使用 Git Bash 执行包含中文的 curl 命令时，可能会遇到 GBK 编码问题。请参阅 [Windows 环境注意事项](#7-windows-环境注意事项) 章节。

本文档提供四类 AI 接口的 curl 测试指令，覆盖非流式和流式请求。

所有命令假设代理运行在 `http://localhost:8080`，使用模型 `mimo-v2.5-pro`。请根据实际配置调整地址和模型名。

协议格式参考：https://docs.poixe.com/cn/api-reference/text-api/openai-completions/overview

## 1. OpenAI Chat Completions

### 1.1 非流式

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": false,
    "temperature": 0.7,
    "max_completion_tokens": 256
  }' | python -m json.tool
```

### 1.2 流式

```bash
curl -s -N -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": true
  }'
```

### 1.3 带工具调用

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "user", "content": "北京今天的天气怎么样？"}
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "Get the current weather for a location",
          "parameters": {
            "type": "object",
            "properties": {
              "location": {"type": "string", "description": "City name"}
            },
            "required": ["location"]
          }
        }
      }
    ],
    "stream": false
  }' | python -m json.tool
```

## 6. 错误场景测试

### 6.1 缺少 model

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "hi"}]}' | python -m json.tool
```

预期：`400 missing_model`

### 6.2 无效 JSON

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d 'not json' | python -m json.tool
```

预期：`400 invalid_json`

### 6.3 未匹配的模型

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "nonexistent-model", "messages": [{"role": "user", "content": "hi"}]}' | python -m json.tool
```

预期：`404 no_channel`

### 6.4 旧配置字段检测

在 `config.yaml` 中故意添加 `wire_api: "chat"`，启动服务。

预期：启动失败，错误信息 `wire_api has been removed, use interface capability URLs instead`

## 7. Windows 环境注意事项

系统提供了一组调试接口，可以获取 curl 命令模板或查看转换后的上游请求，而不实际发送请求。

### 8.1 获取接口模板（GET）

使用 GET 请求获取对应接口的 curl 命令模板：

```bash
# 获取 OpenAI Chat 接口模板
curl http://localhost:8080/curl/v1/chat/completions

# 获取 OpenAI Responses 接口模板
curl http://localhost:8080/curl/v1/responses

# 获取 Anthropic Messages 接口模板
curl http://localhost:8080/curl/v1/messages

# 获取 Gemini Generate Content 接口模板
curl http://localhost:8080/curl/v1beta/models/mimo-v2.5-pro:generateContent
```

### 8.2 查看转换后的上游请求（POST）

使用 POST 请求发送实际请求体，系统会返回转换后的上游 curl 命令：

```bash
# 查看 Chat 请求的上游转发命令
curl -X POST http://localhost:8080/curl/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": false
  }'

# 查看 Messages 请求的上游转发命令
curl -X POST http://localhost:8080/curl/v1/messages \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "max_tokens": 256,
    "messages": [
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": false
  }'
```

### 8.3 使用场景

1. **验证请求格式**：查看系统期望的请求体结构
2. **调试协议转换**：查看从一种协议转换到另一种协议后的请求体
3. **排查上游问题**：查看实际发送到上游的完整请求
4. **测试渠道配置**：验证渠道配置是否正确

## 8. 调试接口（Dry-Run）

### 7.1 问题描述

在 Windows 环境下使用 Git Bash 执行包含中文字符的 curl 命令时，可能会遇到以下错误：

```json
{"error":{"code":"400","message":"Invalid request","param":"400 BAD_REQUEST \"Failed to read HTTP message\"","type":""}}
```

这是因为：
1. Windows 中文环境默认终端编码是 GBK（代码页 936）
2. Git Bash 继承了系统默认编码
3. curl 从命令行接收参数时使用终端编码（GBK）
4. 服务器期望 UTF-8 编码，导致解析失败

### 7.2 解决方案

#### 方案 1：使用 PowerShell（推荐）

```powershell
# 设置控制台编码为 UTF-8
chcp 65001

# 然后执行 curl 命令
curl -s -X POST http://localhost:8080/v1/chat/completions 
  -H "Content-Type: application/json" 
  -d '{"model":"mimo-v2.5-pro","messages":[{"role":"user","content":"用一句话介绍 Go 语言"}]}'
```

#### 方案 2：使用 CMD

```cmd
:: 设置控制台编码为 UTF-8
chcp 65001

:: 然后执行 curl 命令（注意使用双引号）
curl -s -X POST http://localhost:8080/v1/chat/completions ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"mimo-v2.5-pro\",\"messages\":[{\"role\":\"user\",\"content\":\"用一句话介绍 Go 语言\"}]}"
```

#### 方案 3：将请求体写入文件（最可靠）

```bash
# 在 Git Bash 中创建 UTF-8 编码的请求文件
cat > /tmp/request.json << 'EOF'
{
  "model": "mimo-v2.5-pro",
  "messages": [{"role": "user", "content": "用一句话介绍 Go 语言"}]
}
EOF

# 发送文件
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d @/tmp/request.json
```

#### 方案 4：设置 Git Bash 编码

在 Git Bash 中执行：

```bash
# 临时设置
export LANG=zh_CN.UTF-8
export LC_ALL=zh_CN.UTF-8
```

永久设置：编辑 ~/.bashrc 或 ~/.bash_profile，添加上述两行。

#### 方案 5：使用 Python 测试脚本

使用项目提供的 Python 测试脚本，它已正确处理编码问题：

```bash
# 测试所有接口
python tools/test_endpoint.py

# 只测试编码场景
python tools/test_endpoint.py --encoding-only
```

### 7.3 验证当前编码

在 Git Bash 中运行以下命令检查当前编码设置：

```bash
# 查看语言设置
echo $LANG

# 查看区域设置
locale

# 测试终端编码
echo "测试中文" | xxd
```

### 7.4 最佳实践

1. **推荐使用 PowerShell + chcp 65001**：这是最简单可靠的方案
2. **避免在 Git Bash 中直接粘贴包含中文的长命令**：改为写入文件后执行
3. **使用 Python 测试脚本**：tools/test_endpoint.py 已处理各种编码边界情况
4. **检查返回结果**：如果看到乱码或解析错误，首先检查终端编码
