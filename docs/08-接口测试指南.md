# 接口测试指南

本文档提供四类 AI 接口的 curl 测试指令，覆盖非流式和流式请求。

所有命令假设代理运行在 `http://localhost:8080`，使用模型 `mimo-v2.5-pro`。请根据实际配置调整地址和模型名。

## 1. OpenAI Chat Completions

### 1.1 非流式

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": false,
    "temperature": 0.7,
    "max_completion_tokens": 256
  }' | python -m json.tool
```

### 1.2 流式

```bash
curl -s -N -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "user", "content": "用一句话介绍 Go 语言"}
    ],
    "stream": true
  }'
```

### 1.3 带工具调用

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "messages": [
      {"role": "user", "content": "北京今天的天气怎么样？"}
    ],
    "tools": [
      {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "Get the current weather for a location",
          "parameters": {
            "type": "object",
            "properties": {
              "location": {"type": "string", "description": "City name"}
            },
            "required": ["location"]
          }
        }
      }
    ],
    "stream": false
  }' | python -m json.tool
```

## 2. OpenAI Responses

### 2.1 非流式

```bash
curl -s -X POST http://localhost:8080/v1/responses \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "input": "用一句话介绍 Go 语言",
    "instructions": "You are a helpful assistant.",
    "stream": false
  }' | python -m json.tool
```

### 2.2 流式

```bash
curl -s -N -X POST http://localhost:8080/v1/responses \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "input": "用一句话介绍 Go 语言",
    "stream": true
  }'
```

### 2.3 多轮对话

```bash
curl -s -X POST http://localhost:8080/v1/responses \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5-pro",
    "input": [
      {"type": "message", "role": "user", "content": [{"type": "input_text", "text": "Hi"}]},
      {"type": "message", "role": "assistant", "content": [{"type": "output_text", "text": "Hello! How can I help you?"}]},
      {"type": "message", "role": "user", "content": [{"type": "input_text", "text": "用一句话介绍 Go 语言"}]}
    ],
    "stream": false
  }' | python -m json.tool
```

## 5. 跨协议转换测试

以下测试验证系统自动协议转换能力。假设渠道只配置了 `chat_url`，则所有非 Chat 请求都需要经过转换。

### 5.1 Responses -> Chat 转换

```bash
curl -s -X POST http://localhost:8080/v1/responses \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-pro",
    "input": "Say hello in one word",
    "stream": false
  }' | python -m json.tool
```

### 5.2 Messages -> Chat 转换

```bash
curl -s -X POST http://localhost:8080/v1/messages \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-pro",
    "max_tokens": 64,
    "messages": [{"role": "user", "content": "Say hello in one word"}],
    "stream": false
  }' | python -m json.tool
```

### 5.3 Gemini -> Chat 转换

```bash
curl -s -X POST http://localhost:8080/v1beta/models/deepseek-v4-pro:generateContent \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {"role": "user", "parts": [{"text": "Say hello in one word"}]}
    ]
  }' | python -m json.tool
```

### 5.4 Chat -> Messages 转换（流式）

```bash
curl -s -N -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-20250514",
    "messages": [{"role": "user", "content": "Say hello"}],
    "stream": true
  }'
```

## 6. 错误场景测试

### 6.1 缺少 model

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"messages": [{"role": "user", "content": "hi"}]}' | python -m json.tool
```

预期：`400 missing_model`

### 6.2 无效 JSON

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d 'not json' | python -m json.tool
```

预期：`400 invalid_json`

### 6.3 未匹配的模型

```bash
curl -s -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model": "nonexistent-model", "messages": [{"role": "user", "content": "hi"}]}' | python -m json.tool
```

预期：`404 no_channel`

### 6.4 旧配置字段检测

在 `config.yaml` 中故意添加 `wire_api: "chat"`，启动服务。

预期：启动失败，错误信息 `wire_api has been removed, use interface capability URLs instead`
