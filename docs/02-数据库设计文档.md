# resp2chat 数据库设计文档

## 1. 技术选型

| 特性 | 选型 |
|------|------|
| 数据库引擎 | SQLite 3 |
| Go 驱动 | modernc.org/sqlite (纯 Go，无 CGO) |
| 数据文件 | `./data/proxy.db`（默认） |
| 迁移机制 | 启动时自动执行 DDL |

## 2. 表结构

### 2.1 request_logs — 请求日志

记录每一次 API 请求的详细信息。

```sql
CREATE TABLE request_logs (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    request_id      TEXT NOT NULL,           -- 请求唯一 ID
    timestamp       INTEGER NOT NULL,        -- 时间戳 (ms)
    channel_id      TEXT NOT NULL,           -- 渠道 ID
    client_api      TEXT NOT NULL,           -- 客户端 API 类型 ("responses" | "chat")
    upstream_api    TEXT NOT NULL,           -- 上游 API 类型 ("responses" | "chat")
    model           TEXT NOT NULL,           -- 模型 ID
    status_code     INTEGER NOT NULL,        -- HTTP 状态码
    latency_ms      INTEGER NOT NULL,        -- 请求耗时 (ms)
    key_name        TEXT,                    -- 使用的 Key 名称（脱敏）
    error_code      TEXT,                    -- 错误代码
    error_message   TEXT,                    -- 错误信息
    prompt_tokens   INTEGER,                 -- 输入 token 数
    completion_tokens INTEGER,               -- 输出 token 数
    total_tokens    INTEGER,                 -- 总 token 数
    is_stream       INTEGER NOT NULL DEFAULT 0,  -- 是否流式
    request_body    TEXT,                    -- 请求体（可选）
    response_body   TEXT                     -- 响应体（可选）
);
```

**索引**：
```sql
CREATE INDEX idx_logs_timestamp ON request_logs(timestamp DESC);
CREATE INDEX idx_logs_channel ON request_logs(channel_id, timestamp DESC);
CREATE INDEX idx_logs_status ON request_logs(status_code, timestamp DESC);
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 自增主键 |
| request_id | TEXT | 格式 `req_<timestamp>` |
| timestamp | INTEGER | Unix 毫秒时间戳 |
| channel_id | TEXT | 对应配置中的 channel.id |
| client_api | TEXT | 客户端发送的 API 类型 |
| upstream_api | TEXT | 实际调用的上游 API 类型 |
| model | TEXT | 实际发送到上游的模型 ID |
| status_code | INTEGER | 上游返回的 HTTP 状态码 |
| latency_ms | INTEGER | 从接收请求到返回响应的总耗时 |
| key_name | TEXT | 使用的 Key 名称（前4位+***） |
| error_code | TEXT | 代理内部错误代码 |
| error_message | TEXT | 错误详细信息 |

### 2.2 key_stats — Key 统计

持久化每个 Key 的使用统计，重启后恢复。

```sql
CREATE TABLE key_stats (
    channel_id      TEXT NOT NULL,           -- 渠道 ID
    key_name        TEXT NOT NULL,           -- Key 名称
    request_count   INTEGER NOT NULL DEFAULT 0,  -- 总请求次数
    error_count     INTEGER NOT NULL DEFAULT 0,  -- 总错误次数
    total_latency_ms INTEGER NOT NULL DEFAULT 0, -- 总延迟
    last_error      TEXT,                    -- 最后错误信息
    last_error_time INTEGER,                 -- 最后错误时间
    last_success_time INTEGER,               -- 最后成功时间
    paused          INTEGER NOT NULL DEFAULT 0,  -- 是否暂停
    pause_until     INTEGER,                 -- 暂停到期时间
    PRIMARY KEY (channel_id, key_name)
);
```

## 3. 数据操作

### 3.1 插入日志

```go
func (db *DB) InsertLog(reqID, channelID, clientModel, upstreamModel string, status int, latencyMs int64, key, errorCode, errorMsg string)
```

### 3.2 查询日志

```go
func (db *DB) QueryLogs(channelID string, statusMin, statusMax int, from, to int64, limit, offset int) ([]LogEntry, error)
```

支持的过滤条件：
- `channelID`：按渠道过滤
- `statusMin/statusMax`：按状态码范围过滤
- `from/to`：按时间范围过滤
- `limit/offset`：分页

### 3.3 删除日志

```go
func (db *DB) DeleteLogsBefore(timestamp int64) (int64, error)
```

### 3.4 数据库维护

```go
func (db *DB) GetLogCount() (int64, error)
func (db *DB) Vacuum() error
```

## 5. 数据安全

- `key_name` 字段存储脱敏后的 Key（前4位+***+后4位）
- `request_body` 和 `response_body` 可选记录（通过 `log_request_body` 配置控制）
- SQLite WAL 模式，支持并发读写
