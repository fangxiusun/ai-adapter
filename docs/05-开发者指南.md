## 3. 代码规范

### 3.1 包组织

- `cmd/`：入口程序
- `internal/`：内部包（不可被外部导入）
- 每个包职责单一：
  - `translate/`：仅做协议翻译
  - `channel/`：仅做渠道和Key管理
  - `proxy/`：仅做请求路由和协调

### 3.2 命名规范

- **包名**：小写单词（如 `translate`, `channel`）
- **文件名**：小写 + 下划线（如 `keypool.go`）
- **函数名**：PascalCase（如 `ReqToChat`）
- **变量名**：camelCase（如 `upstreamModel`）
- **常量**：PascalCase 或 camelCase

### 3.3 错误处理

```go
// 返回错误而非 panic
func doSomething() error {
    if err := validate(); err != nil {
        return fmt.Errorf("validate: %w", err)
    }
    return nil
}

// 调用方处理错误
if err := doSomething(); err != nil {
    log.Error("failed", "error", err)
}
```

### 3.4 并发安全

- `KeyPool` 使用 `sync.RWMutex` 保护共享状态
- `ChannelManager` 使用 `sync.RWMutex` 保护渠道映射
- `DB` 使用 `sync.RWMutex` 保护数据库连接

## 4. 核心概念

### 4.1 翻译方向

每个渠道配置 `wire_api` 决定翻译方向：

```go
// wire_api="chat"：上游说 Chat Completions
// 客户端发 Responses → 翻译为 Chat → 发给上游
// 客户端发 Chat → 直接透传

// wire_api="responses"：上游说 Responses
// 客户端发 Responses → 直接透传
// 客户端发 Chat → 翻译为 Responses → 发给上游
```

### 4.2 Key 管理

Key 池支持四种轮转策略：

```go
// round-robin：轮询
index := atomic.AddUint64(&kp.counter, 1)
return available[index%len(available)]

// random：随机
return available[rand.Intn(len(available))]

// least-errors：最少错误
best := available[0]
for _, k := range available[1:] {
    if k.State.ErrorCount < best.State.ErrorCount {
        best = k
    }
}

// least-latency：最低延迟
best := available[0]
for _, k := range available[1:] {
    if k.State.AvgLatencyMs() < best.State.AvgLatencyMs() {
        best = k
    }
}
```

### 4.3 Fan-out

```go
// 获取 N 个 key
keys := ch.KeyPool().GetN(count)

// 并发发送
for _, key := range keys {
    go func(k *KeyEntry) {
        result := ch.sendRequest(ctx, k, req)
        results <- result
    }(key)
}

// wait_all=false：第一个成功返回
for result := range results {
    if result.Error == nil && result.StatusCode < 300 {
        return result
    }
}
```

### 4.4 流式翻译

流式翻译使用状态机模式：

```go
type streamState struct {
    activeKind   string  // "reasoning" | "message" | null
    activeItemID string
    activeBuffer string
    toolCalls    map[int]*toolCallState
}

// 每个 chunk 进来时：
func processChunk(chunk) {
    if chunk.delta.reasoning_content {
        if state.activeKind != "reasoning" {
            openReasoning()  // 关闭当前 item，开启新的
        }
        state.activeBuffer += chunk.delta.reasoning_content
        emit("response.reasoning_summary_text.delta", ...)
    }
    if chunk.delta.content {
        if state.activeKind != "message" {
            openMessage()
        }
        state.activeBuffer += chunk.delta.content
        emit("response.output_text.delta", ...)
    }
}
```
