# resp2chat 项目概览

## 1. 项目定位

resp2chat 是一个用 Go 编写的高性能 LLM API 协议转换代理，核心功能是在 **OpenAI Responses API** 和 **Chat Completions API** 之间做实时双向翻译。

```
客户端 (Codex/任意HTTP) → resp2chat → 渠道选择 → Key池 → 上游LLM
                              ↓
                        协议翻译层
                   (Responses ↔ Chat Completions)
```

## 2. 核心能力

| 能力 | 说明 |
|------|------|
| 双向协议翻译 | Responses → Chat Completions / Chat → Responses |
| 多渠道管理 | 每个渠道独立配置上游地址、Key、模型、策略 |
| 多Key轮转 | round-robin / random / least-errors / least-latency |
| Key健康追踪 | 自动暂停连续错误Key，自动恢复 |
| Fan-out | 多Key并发请求，返回第一个成功结果 |
| 流式翻译 | SSE 流式响应的完整翻译 |
| 嵌入式Web UI | Alpine.js 管理面板 |
| 结构化日志 | 可配置级别、请求体记录 |
| SQLite持久化 | 请求日志和Key统计重启后保留 |

```
├── internal/
│   ├── config/                  # YAML配置
│   ├── translate/               # 协议翻译层
│   │   ├── types.go             # 类型定义
│   │   ├── req.go               # 请求翻译
│   │   ├── resp.go              # 响应翻译
│   │   ├── stream.go            # 流式翻译
│   │   └── tools.go             # 工具映射
│   ├── channel/                 # 渠道管理
│   │   ├── channel.go           # 渠道抽象
│   │   ├── keypool.go           # Key池
│   │   ├── keystate.go          # Key状态
│   │   └── fanout.go            # 并发请求
│   ├── proxy/                   # 代理核心
│   │   └── handler.go           # 请求处理
│   ├── db/                      # 数据库
│   │   └── db.go                # SQLite
│   ├── web/                     # Web UI
│   │   ├── handler.go           # Admin API

```

## 4. 技术栈

| 组件 | 技术 |
|------|------|
| 语言 | Go 1.21+ |
| HTTP路由 | net/http + chi |
| 配置 | YAML (gopkg.in/yaml.v3) |
| 数据库 | SQLite (modernc.org/sqlite, 纯Go) |
| 嵌入式前端 | Go embed.FS + Alpine.js |
| 日志 | log/slog (标准库) |

## 5. 与 mimo2codex 的关系

- **独立项目**：代码完全重写，仅参考翻译逻辑
- **互补共存**：Go 版本作为高性能替代
- **翻译逻辑一致**：保持与 mimo2codex 相同的协议翻译能力
